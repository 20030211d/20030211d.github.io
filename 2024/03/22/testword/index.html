实验题目：利用主流云平台搭建个人博客或网站
实验学时：2 实验日期： 2024.3.22
实验目的：熟悉使用主流云平台并搭建个人博客或者网站。
具体包括：
参考方案：基于主流云平台，设计、实现个人博客或者网站的搭建，撰写实验报告（附带网站链接并可以访问），并在网站上呈现此次实验报告。

硬件环境： 
联网的计算机一台

软件环境：
Windows or Linux

实验步骤与内容：
1.准备一个github账号，并安装好git和NodeJS；
2.创建仓库，仓库名的格式为<用户名>.github.io,之后点击创建仓库，创建一个新文件作为网站主页，文件名必须为index.html，填写完内容后在Github Page中找到主页地址为https://20030211d.github.io/；
3.安装Hexo，打开终端，输入npm install -g hexo-cli，安装完之后，创建一个项目 hexo-blog 并初始化，依次输入hexo init hexo-blog、cd hexo-blog、npm install，本地启动依次输入hexo g、hexo server，最终根据终端的输出内容查看网址http://localhost:4000的内容；

4.创建文章：如下修改 Hexo 博客目录中的 _config.yml：post_asset_folder: true，执行如下命令创建一篇新文章：hexo new post testword；执行完成后在source_posts目录下生成了一个md文件和一个同名的资源目录(用于存放图片)，在testword.md中添加内容，引用图片方式是使用下列的标签插件：，之后再次本地启动，再次访问http://localhost:4000；
5.发布到GitHub Pages,安装hexo-deployer-git:npm install hexo-deployer-git –save,修改根目录下的 _config.yml，配置 GitHub 相关信息:
deploy:
type: git
repo: https://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@github.com/20030211d/20030211d.github.io.git
branch: main
6.部署到GitHub:hexo g -d,浏览器访问 https://20030211d.github.io/，部署成功;

结论分析与体会：

1.本次实验中选择的github可以被归类为一种软件即服务（SaaS）。作为一个基于云的代码托管平台，GitHub 提供了基于订阅的服务，让用户可以在互联网上访问和使用该平台的各种功能，而无需在本地安装或维护相关软件。用户可以通过网页界面或者命令行工具来管理他们的代码仓库，进行协作开发，执行版本控制等操作。因此，根据 SaaS 的定义，GitHub 可以被认为是一种软件即服务。
2.尽管 GitHub 本身并不提供完整的平台即服务（PaaS）功能，但它可以与各种 PaaS 提供商（如Heroku、Azure、AWS等）集成，以便开发者能够将他们的应用程序部署到云平台上。
3.阿里云提供从Iaas到SaaS都有的服务。阿里云作为典型的IaaS+PaaS层的云计算服务供应商，目前提供的产品服务主要有弹性计算、存储、数据库、安全、大数据等十三大类。阿里云应该是把IaaS、PaaS、SaaS三种模型结合在一起的，一层建立在另一层之上，利用PaaS环境提供的已就绪环境来开发和部署它自己的SaaS云服务，然后以商业产品的形式提供云服务。

体会：本实验选择使用了github来作为云平台来实现个人博客网站的搭建，采用了csdn上的一篇文章来搭建平台，可能最终制作出来的博客网站比较简单，没有很多功能，上传的实验报告写的也不是很详细，因为做实验的时候没有记录图片；最开始的时候是尝试使用阿里云来做实验的，但是配好弹性云服务器后，在远程终端上发现下载mysql需要很长时间，最后选择github来作为最后实验结果，可能在此处的实验报告上就不讲述阿里云的实验结果了，但是最后交上去的实验报告上可能会附有阿里云的实验结果，总的来说，本次实验的难点主要在于环境配置的困难与上传内容的麻烦，其他有教程都比较简单。
